# SnapBar ⚡ —— 让重复不再发生，让灵感随触即达

## 💡 开发初衷：为什么我们需要 SnapBar？

你是否也经历过这样的时刻？

*   写邮件时，那段礼貌的开头和结尾，每天要敲十几遍。
*   写代码时，那个常用的 Boilerplate 模版，每次都要去旧项目里翻找复制。
*   使用 AI 工具时，那几个精心调教的 Prompt，总是需要打开记事本、Ctrl+C、Ctrl+V……

**“重复”是效率的杀手。**

我们发现，在日常工作流中，有大量的时间被浪费在了“寻找”和“重复输入”上。虽然市面上有剪贴板管理工具，但它们往往太重、太复杂，或者无法针对“固定内容”进行高效管理。

**SnapBar 因此诞生。**

它的理念很简单：**把你需要高频使用的内容，变成手边最顺手的工具栏。** 它像一个隐形的助手，安静地悬浮在屏幕边缘，当你需要时，只需轻轻一点，内容即刻上屏。我们希望通过 SnapBar，帮你省去那几秒钟的切换和查找，让你的思路不再被打断，让工作流真正“流”动起来。

## 🚀 SnapBar 能为你做什么？

SnapBar 是一个极其轻量的桌面悬浮工具，专为管理和快速输入文本片段而生。

### 1. **悬浮置顶，即点即用**
它永远悬浮在你的桌面最顶层（可拖拽移动），无论你正在使用 Word、VS Code 还是浏览器，SnapBar 就在手边。
*   **左键点击**：复制内容到剪贴板，并自动隐藏窗口，不遮挡视线。
*   **右键点击**：快速编辑内容。

### 2. **不仅是剪贴板，更是你的“第二大脑”**
不同于普通的剪贴板记录历史，SnapBar 管理的是你**精心沉淀的知识**。
*   **AI 提示词 (Prompts)**：保存你最满意的 ChatGPT/Midjourney 指令。
*   **代码片段 (Snippets)**：常用的 SQL 查询、Linux 命令、代码模版。
*   **办公语料**：邮件回复模版、客服话术、常用链接。

### 3. **极简操作，功能强大**
我们深知“简单”的重要性，但在简单之下，我们藏入了强大的功能：
*   **🔐 隐私安全**：支持 AES-GCM 加密。哪怕是 API Key 这种敏感信息，也可以放心存入。设置密码后，只有你能查看。
*   **💾 永不丢失**：本地持久化存储，支持一键备份为 JSON 文件。
*   **🔄 批量分享**：觉得好用的提示词想发给同事？支持导出为 TXT 文件，对方一键导入即可使用。

### 4. **完全个性化**
*   **无级缩放**：屏幕小？调小点。老花眼？拉大点。滑动条支持 50% - 150% 任意缩放。
*   **多彩主题**：心情不好换个色，支持自定义任意背景颜色。

## 📖 快速上手指南

### 第一步：安装与启动
下载并安装 `SnapBar`。启动后，你会看到一个精致的悬浮条出现在屏幕上。

### 第二步：添加你的第一个“快捷键”
1.  点击悬浮条上的 **`+`** 号。
2.  输入标题（如“周报模版”）和内容。
3.  点击保存。
现在，你的第一个快捷按钮就诞生了！

### 第三步：使用它
在任何输入框中，点击这个按钮。内容已经复制进剪贴板，你只需要 `Ctrl+V` (粘贴) 即可。

### 第四步：进阶玩法
*   **批量导入**：
    在“设置”中选择“Import TXT”，选择一个包含如下内容的文本文件，瞬间批量添加几十条提示词：
    ```text
    {name：学术润色，content：请帮我把这段话修改得更学术化...}
    {name：代码解释，content：请逐行解释下面这段代码的逻辑...}
    ```
*   **开启加密**：
    在“设置”中设置一个密码。下次启动软件时，必须输入密码才能看到你的秘密库。

## 🛠️ 技术栈

本项目采用现代化的前后端分离架构，确保高性能与良好的开发体验：

### 前端 (Frontend)
*   **语言**：TypeScript
*   **框架**：React 19
*   **构建工具**：Vite
*   **UI 样式**：TailwindCSS 4
*   **图标库**：Lucide React
*   **路由**：React Router DOM

### 后端 (Backend / Native)
*   **框架**：Tauri v2
*   **语言**：Rust (2021 edition)
*   **关键库**：
    *   `enigo`：用于自动化输入模拟
    *   `windows`：用于 Windows API 交互
    *   `tauri-plugin-store`：用于数据持久化

## 🍿 原理科普：它是怎么工作的？（给非技术人员）

如果把 SnapBar 比作一家**餐厅**：

1.  **前端 (React)** 就像是**精致的菜单和帅气的服务员**。
    *   它负责“长得好看”和“接待客人”。你看到的按钮、动画、颜色，都是由它画出来的。
    *   它和你平时访问的网站用的是一样的技术，所以界面可以做得非常现代、流畅。

2.  **后端 (Rust)** 就像是**后厨里不仅手速快、还极其严谨的大厨**。
    *   服务员（前端）是不能直接进仓库拿菜的。
    *   当你点击“保存”或者“粘贴”时，服务员会把需求传给后厨。
    *   **Rust** 这位大厨负责真正干重活：把你的数据加密锁进保险柜（硬盘），或者指挥操作系统帮你按下键盘。它以安全和高性能著称，保证了软件既快又稳，不会动不动就“把厨房炸了”（崩溃）。

3.  **Tauri** 就像是**这家餐厅的建筑本身**。
    *   它把“服务员”和“大厨”包装在一起，变成一个完整的门店（桌面软件）。
    *   普通的网页被关在浏览器这个“沙盒”里，不仅不能随便读写文件，更不能控制键盘。Tauri 给了这个网页一把“万能钥匙”，让它能安全地调用电脑的底层能力。

**一句话总结**：
SnapBar 本质上是一个**拥有了系统级超能力的网页**。既保留了网页的轻盈美观，又拥有了原生软件的强大功能。
